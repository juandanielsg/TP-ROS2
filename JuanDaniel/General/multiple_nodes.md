# Multiple Nodes in One Package

**What you will learn:** how to add several ROS2 nodes to the same Python package so they can all be run with `ros2 run`, without creating a separate package for each one.

---

## Prerequisites

- A working ROS2 Python package (see Session 2)
- Familiarity with the `setup.py` file generated by `ros2 pkg create`

---

## 1. How It Works

When `ros2 pkg create` generates a package, it registers one node as an entry point in `setup.py`:

```python
entry_points={
    'console_scripts': [
        'turtlebot4_first_python_node = turtlebot4_python_tutorials.turtlebot4_first_python_node:main',
    ],
},
```

Each line in `console_scripts` maps a command name to a `module:function` pair. Adding a new node is just a matter of:

1. Creating a new `.py` file in the package directory.
2. Adding a new line to `console_scripts`.
3. Rebuilding.

---

## 2. File Structure

Place every node file inside the inner package directory (the one that contains `__init__.py`):

```
turtlebot4_python_tutorials/
├── package.xml
├── setup.py
├── setup.cfg
└── turtlebot4_python_tutorials/
    ├── __init__.py
    ├── turtlebot4_first_python_node.py   ← existing node
    ├── odom_reader.py                    ← new node
    └── battery_monitor.py               ← another new node
```

Each file must define a `main()` function, which is the entry point ROS2 will call.

---

## 3. Register the New Nodes

Open `setup.py` and add one line per new node to `console_scripts`:

```python
entry_points={
    'console_scripts': [
        'turtlebot4_first_python_node = turtlebot4_python_tutorials.turtlebot4_first_python_node:main',
        'odom_reader        = turtlebot4_python_tutorials.odom_reader:main',
        'battery_monitor    = turtlebot4_python_tutorials.battery_monitor:main',
    ],
},
```

The format for each entry is:

```
'<command_name> = <package_name>.<module_name>:<function_name>'
```

- `command_name`: what you type after `ros2 run <package>`
- `package_name`: the Python package (same as the directory containing `__init__.py`)
- `module_name`: the filename without `.py`
- `function_name`: usually `main`

---

## 4. Build and Run

Rebuild the package to register the new entry points:

```bash
cd ~/turtlebot4_ws
colcon build --symlink-install --packages-select turtlebot4_python_tutorials
source install/local_setup.bash
```

> Even with `--symlink-install`, you must rebuild whenever you edit `setup.py`. Changes to `.py` node files themselves do not require a rebuild.

Then run any node by its command name:

```bash
ros2 run turtlebot4_python_tutorials odom_reader --ros-args -r __ns:=/tbot<N>
ros2 run turtlebot4_python_tutorials battery_monitor --ros-args -r __ns:=/tbot<N>
```

---

## 5. Sharing Code Between Nodes

If several nodes use the same helper (e.g. the `quaternion_to_yaw` function from Session 4), extract it into a separate file instead of copying it:

```
turtlebot4_python_tutorials/
└── turtlebot4_python_tutorials/
    ├── __init__.py
    ├── utils.py                  ← shared helpers
    ├── odom_reader.py
    └── battery_monitor.py
```

Import it like any regular Python module:

```python
from turtlebot4_python_tutorials.utils import quaternion_to_yaw
```

Do **not** add `utils.py` to `console_scripts`; it is a library file, not a node.

---

## Troubleshooting

| Symptom | Likely cause |
|---------|--------------|
| `ros2 run` says the executable is not found | New entry not in `console_scripts`, or package not rebuilt after editing `setup.py` |
| `ModuleNotFoundError` at runtime | File is not inside the inner package directory, or `install/local_setup.bash` not sourced after rebuild |
| Changes to the node code have no effect | With `--symlink-install` this should not happen; if it does, check that the symlink points to the right file with `ls -la install/` |
